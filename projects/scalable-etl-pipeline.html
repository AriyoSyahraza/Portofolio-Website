Pipeline Architecture

  • 1. Extract (Python): Script custom yang mengekstrak log dari Mock APIs dan Database operasional.
  • 2. Load (PostgreSQL): Data mentah disimpan utuh di *landing zone* untuk referensi historis.
  • 3. Transform (dbt): Transformasi SQL di dalam Warehouse. Mengubah data mentah menjadi *Fact & Dimension Tables*, menangani missing value, dan melakukan standarisasi data.
  • 4. Orchestration (Airflow): Merangkai seluruh *tasks* menggunakan DAG di Airflow dengan *retry logic* otomatis bila API *down*.

Why It Stands Out

Penggunaan dbt (data build tool) dan Airflow mencerminkan praktik nyata di industri saat ini. Ini memperlihatkan kemampuan saya dalam mengelola *data lineage* dan *idempotency* yang lebih berharga daripada script Pandas tunggal.
