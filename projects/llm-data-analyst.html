Problem Statement & Objective

Menganalisis database besar seringkali membutuhkan waktu lama dari Data Analyst untuk sekadar menulis SQL dasar dan membuat rangkuman. Ini menyebabkan bottleneck dalam pengambilan keputusan operasional.

Objective: Membangun AI Agent yang dapat menerima pertanyaan dalam bahasa natural (Text-to-SQL), melakukan koreksi otomatis jika query gagal, lalu mengeksekusinya untuk mensintesis insight bisnis langsung kepada end-user.

Workflow & Process (LangGraph)

01
User Input & Context Retrieval

Agent membaca prompt user dan menarik skema tabel database (DDL) untuk memahami struktur data.

02
SQL Generation Agent

LLM men-generate sintaks SQL yang paling efisien berdasarkan dialek database (PostgreSQL).

03
Execution & Self-Correction (ReAct)

Mengeksekusi SQL. Jika ada error, error log diumpankan kembali ke LLM untuk direvisi secara otonom (maksimal 3 iterasi).

04
Insight Synthesis Agent

Hasil query raw diubah menjadi narasi bahasa manusia yang menyertakan actionable insight bisnis.

Why It Stands Out

Project ini membuktikan kapabilitas Agentic AI Engineering. Tidak seperti sekadar memanggil API LLM (single-turn), arsitektur LangGraph memungkinkan AI memiliki *state*, berpikir matematis, dan mengoreksi kesalahannya sendiri, menunjukkan pemahaman mendalam tentang ekosistem LLM modern.
