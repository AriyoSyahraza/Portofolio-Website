Algorithm

Problem & Objective

Katalog e-commerce yang sangat besar seringkali menyulitkan pelanggan menemukan barang spesifik. Pendekatan rule-based tidak cukup untuk menangani variasi *taste* tiap pengguna secara personal.

Objective: Merancang ML system (Collaborative Filtering) yang mampu mempelajari relasi implicit (klik, add-to-cart, view) dan men-deploy-nya agar bisa diakses secara *real-time* oleh frontend aplikasi e-commerce.

Architecture & Deployment

  • Model Training: Menggunakan library `implicit` (Alternating Least Squares) yang efisien untuk sparse matrices.
  • API Layer: FastAPI dipilih karena performa asynchronous-nya yang setara Node.js / Go.
  • Caching Layer: Hasil inferensi untuk item/user yang populer di-cache ke dalam Redis untuk memotong latensi hingga < 50ms.
  • Containerization: Seluruh aplikasi dibungkus dalam kontainer Docker untuk memastikan skalabilitas (Production-ready).

Why It Stands Out

Fokus project ini bergeser dari sekadar "tuning model" ke *MLOps & Software Engineering*. Kemampuan merangkai Pipeline Model ke bentuk API yang ringan, cepat, dan siap di-deploy adalah nilai krusial yang dicari oleh Tech Recruiter modern.
